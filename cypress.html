Cypress installation

Pre-conditions

  1. Node version manager NVM here
  2. Open a terminal in you project folder
  3. type:   npm init and complet the require fields
  4. Install cypress (inside your project)--> npm install cypress --save-dev
  5. Check cypress version --> npm cypress -version
  6. Execute to open cypres: ./node_modules/.bin/cypress open
  7. Open cypress (inside your project) --> npx cypress open
  8. Once cypress is open, select e2e testing, continue
  9. add scripts to run in your package.json file
  10. "cy.chrome": "cypress run -b chrome --reporter mochawesome --spec 'cypress/e2e/**'",
  11. "cy.open": "cypress open"
  12. In order to add cucumber:
  13. https://github.com/badeball (plugin owner)
  14. type inside your project: npm i -D cypress @bahmutov/cypress-esbuild-preprocessor esbuild
  15. also the cucumber pre-procesor: npm install @badeball/cypress-cucumber-preprocessor
  16. Test cases folder --> e2e
  17. Cypress test cases extensions --> cy.js
  18. Check cypress documentation:   Cypress documentation
  19. Download cypress snippet for visual code

In order to update cypress

  1. Open cypress project
  2. inside the project, search for the promp update
  3. copy the script, close the project and paste it in the terminal
  4. script example: npm install -D cypress@12.12.0

Tips

Filters allows you to filter an element by an state
  • enabled
  • disabked
  • checked
  • empty
  • visible
  • Usage example:   cy.get('.checkbox':checked)
Usefully assertions
  • should.('exist') -> To make sure that the element exist
  • should.('not.exist') -> To make sure that the element do not exist
  • should.('be.checked') -> To make sure that the box or radio button is checked
  • should.('be.disabled') -> To make sure that the element is disabled
  • .should('have.text','textElement')-> Search for the text of the element
Usefully functions
  • cy.log("text for log in the test execution")-> works for write something between the cases
  • .pause() pause the execution after a instruction, for resume click the play in the cypress runner
  • .debug() use after a instruction, this debug information can be reach in the devtools, open it in the devtools from the cypress runner
  • You can use alias to objects in order to call it in another cases
  • You can wait in milliseconds for a control with cy.wait(1000)
  • In order to avoid cases with multiple tabs in one flow use:   .invoke('removeAttr', 'target')

most used command in cypress

  • .check() -> click on a checkbox
  • .uncheck() -> unclick on a checkbox
  • .select('objectText') -> selects a item from a dropdown list
  • .select('1') -> selects the position of an item from a dropdown list (starts from 1)
  • .clear() -> clear the text that you previously sent
  • .click({force: true}) -> forces a click even if the element is hidden
  • .rightclick() -> press the right mouse button on a element
  • .invoke("trigger,contextmenu") -> triggers a pop up, context menu
  • cy.on("window:alert",(alert) =>{ expect(alert).to.equal("AlertMessage") })

Validations

  • You can add an and to a validation for validate more than one parameter
  • Implicit validation .should()
  • Explicit validation expect().to.equals()

  • You can set no proxy configuration if your environment test have a proxy on it

  • in the package.json file add this array to set no proxy for environments
